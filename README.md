# Antigravity Rules 🪐

**「人間を、最上位の意思決定だけに集中させる。」**
AIエージェントの能力を120%引き出し、開発プロセスを自律化するための最強のルールセットです。

## 🚀 クイックスタート (Quick Start)

1.  **ダウンロード**: **[AntigravityKit.zip](https://github.com/JyozukaTukasa/AntigravityRules/raw/main/AntigravityKit.zip)** をダウンロードします。
2.  **導入**: あなたのプロジェクトの **ルートディレクトリ** に展開します。
    - `.agent/`, `.github/`, `ANTIGRAVITY_MANUAL.md`, `GEMINI.md` が配置されます。
3.  **開始**: AIエージェントに「`@[/onboard]` して」と話しかけてください。

---

## 💡 なぜ Antigravity を導入するのか？

単なるコード規約ではありません。開発の「質」と「速度」を物理的に底上げするシステムです。

### 1. 「考える力」の強化（設計・要件定義）
- **迷わない要件定義**: `@[/spec-phase]` で、AIがインタビュー形式で仕様を引き出します。
- **セキュリティ・バイ・デザイン**: タスク作成時に「攻撃者の視点」での分析をAIに強制し、脆弱性を未然に防ぎます。

### 2. 「安全」を標準装備（セキュリティ・品質）
- **RLS (Row Level Security) の徹底**: DB操作時にポリシーの有無を自動チェック。実装漏れを許しません。
- **10層のガードレール**: セキュリティ、パフォーマンス、SEO、法的要件まで、AIが多角的に自動監査します。

### 3. 「認知負荷」の劇的ダウン（効率化）
- **一貫した命名・コメント**: `is/has`, `handle/on` などの厳格なルールで、他人のコードを読むストレスをゼロにします。
- **クリーンな履歴**: Gitのブランチ管理やマージルールを自動化し、GitHubの履歴を常に美しく保ちます。

## 🛠 魔法のコマンド (Workflows)

| コマンド | 用途 | メリット |
| :--- | :--- | :--- |
| `@[/onboard]` | プロジェクト参画 | コンテキストを一瞬で同期し、即戦力化。 |
| `@[/ship]` | 最終全件監査 | 脆弱性・バグ・速度低下をマージ前に全検知。 |
| `@[/consult]` | 能動的な提案 | 指示を待たず、AIがプロジェクトの改善点を提案。 |
| `@[/sync]` | 文書・コード同期 | 仕様書とコードの「ズレ」を自動で発見・修正。 |
| `@[/review]` | スマートレビュー | コミット/ブランチの差分を自動判定し、核心を突く指摘。 |
